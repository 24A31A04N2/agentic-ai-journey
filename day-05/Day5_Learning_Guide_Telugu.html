HTTP Fundamentals — Agent Tool Call ఎలా Work అవుతుంది?
🇮🇳 Telugu లో Explain
మీరు Swiggy లో food order చేస్తారు కదా? మీరు phone లో "Order" button press చేస్తే — అది ఒక HTTP Request server కి వెళ్తుంది. Server food prepare చేసి "Order confirmed" అని HTTP Response పంపుతుంది.

AI Agent కూడా exactly ఇలాగే work చేస్తుంది! Agent ఒక tool use చేయాలనుకుంటే — అది ఒక HTTP request పంపుతుంది. Tool response వచ్చాక agent next step decide చేస్తుంది.
🎯 Real-Life Analogy
HTTP Request = మీరు restaurant లో waiter కి order చెప్పడం
HTTP Response = Waiter food తీసుకొచ్చి ఇవ్వడం
Headers = "Table number 5, VIP customer" అని extra info చెప్పడం
Body = Actual order — "2 biryanis, 1 raita"

HTTP Request Anatomy

ప్రతి agent tool call ఇలా కనిపిస్తుంది:

# Agent "search" tool call = ఈ HTTP request! POST https://api.openai.com/v1/chat/completions # Headers — extra information Authorization: Bearer sk-proj-your-key-here Content-Type: application/json # Body — actual data (JSON format లో) { "model": "gpt-4o", "messages": [ {"role": "user", "content": "Explain agents"} ] }
🇮🇳 Line-by-Line Explanation
POST — Server కి data పంపుతున్నాం (order place చేస్తున్నాం)
URL — ఏ server కి పంపాలి? OpenAI server
Authorization — "నా API key ఇది, నన్ను allow చెయ్యి" (VIP pass లాంటిది)
Content-Type — "నేను JSON format లో data పంపుతున్నా"
Body — Actual question: "Explain agents" అని GPT-4o ని అడుగుతున్నాం

Status Codes — Server ఏం చెప్తుంది?

🇮🇳 Telugu లో
Status code అంటే — server response లో "ఏం జరిగింది" అని చెప్పే number. Waiter "Ready sir" అంటే 200, "Item లేదు" అంటే 404!
Code Name Telugu Meaning Agent ఏం చేయాలి?
200 OK అంతా బాగుంది, result వచ్చింది ✅ Result process చేయి
201 Created New item create అయింది ✅ New resource save చేయి
400 Bad Request Request తప్పుగా ఉంది ❌ Request fix చేసి retry
401 Unauthorized API key wrong / expired 🔑 Key check చేయి
403 Forbidden Permission లేదు 🚫 Access level check చేయి
404 Not Found ఆ page/resource లేదు 🔍 URL correct గా ఉందా check
429 Rate Limited చాలా fast గా requests పంపుతున్నావ్ 🐌 RETRY with backoff!
500 Server Error Server crash అయింది 💥 RETRY — server side problem
503 Service Unavailable Server busy/down 🔧 RETRY — కొంచెం wait చేయి
# Python లో status codes handle చేయడం response = await client.get("/search") if response.status_code == 200: # Success! Data use చేయి data = response.json() elif response.status_code == 429: # Rate limited! Wait చేసి retry చెయ్యి retry_after = response.headers.get("Retry-After", 5) await asyncio.sleep(int(retry_after)) # Retry the request... elif response.status_code >= 500: # Server error! Retry చెయ్యి raise RetryableError("Server down") else: # Client error (400-499) — fix చేయాలి raise ValueError(f"Error: {response.status_code}")
💡 Important Rule
429 (Rate Limited) — ఇది error కాదు! Server "slow down" అని signal ఇస్తోంది. Production agents ఈ signal handle చేయాలి — లేకపోతే crash అవుతాయి!

Retry with Exponential Backoff — API Fail అయితే ఏం చేయాలి?
🇮🇳 Telugu లో Explain
మీరు friend కి phone చేస్తున్నారు — busy వస్తోంది. ఏం చేస్తారు?

❌ Wrong way: వెంటనే మళ్ళీ call, మళ్ళీ call, మళ్ళీ call... (irritating!)
✅ Right way: 1 sec wait → call → busy → 2 sec wait → call → busy → 4 sec wait → call → answer! 🎉

ఇది Exponential Backoff! ప్రతిసారి wait time double అవుతుంది. ఇలా చేస్తే server కి rest ఇస్తున్నాం, eventually response వస్తుంది.

Exponential Backoff ఎలా Work అవుతుంది?

Formula: delay = base_delay × 2^attempt × random_jitter

Attempt 0: 1.0 × 2⁰ = 1.0s █
Attempt 1: 1.0 × 2¹ = 2.0s ██
Attempt 2: 1.0 × 2² = 4.0s ████
Attempt 3: 1.0 × 2³ = 8.0s ████████
Attempt 4: 1.0 × 2⁴ = 16.0s ████████████████
Attempt 5: 1.0 × 2⁵ = 32.0s ████████████████████████████████

+ Jitter (random variation) add చేస్తాం so that
1000 agents ఒకేసారి retry చేయకుండా!

Python Implementation

from dataclasses import dataclass import random, asyncio @dataclass class RetryConfig: """Retry settings — ఎన్నిసార్లు try చేయాలి, ఎంత wait చేయాలి""" max_retries: int = 3 # Maximum 3 retries base_delay: float = 1.0 # Starting wait: 1 second max_delay: float = 30.0 # Maximum wait: 30 seconds backoff_factor: float = 2.0 # Double every time jitter: bool = True # Add randomness def get_delay(self, attempt: int) -> float: """ప్రతి attempt కి delay calculate చేయడం""" # base × 2^attempt = exponential growth delay = min( self.base_delay * (self.backoff_factor ** attempt), self.max_delay, # 30s కంటే ఎక్కువ wait చేయకు ) if self.jitter: # Random variation add చేయి (0.5x to 1.5x) delay *= random.uniform(0.5, 1.5) return round(delay, 2)
🇮🇳 Code Explanation
max_retries = 3 — Maximum 3 సార్లు try చేస్తాం
base_delay = 1.0 — First time 1 second wait
backoff_factor = 2.0 — ప్రతిసారి delay double అవుతుంది (1→2→4→8)
max_delay = 30.0 — ఎంత delay అయినా 30 seconds కంటే ఎక్కువ wait చేయం
jitter = True — Random variation add చేస్తాం (thundering herd problem avoid కోసం)

@with_retry Decorator — Automatic Retry

def with_retry(config=None): """ ఈ decorator ఏ function కైనా add చేస్తే automatic గా retry logic వస్తుంది! """ if config is None: config = RetryConfig() def decorator(func): async def wrapper(*args, **kwargs): for attempt in range(config.max_retries + 1): try: result = await func(*args, **kwargs) # 429 or 500 వస్తే retry చెయ్యి if result.is_retryable: raise RetryableError(...) return result # Success! except RetryableError: if attempt < config.max_retries: delay = config.get_delay(attempt) print(f"Retry {attempt+1}...") await asyncio.sleep(delay) else: raise # అన్ని retries fail! return wrapper return decorator # Usage — just add @with_retry()! @with_retry() async def search(query): # ఈ function automatically retry అవుతుంది response = await client.get(f"/search?q={query}") return response
🎯 Real-Life Analogy
Decorator = Phone case లాంటిది. Phone (function) ని protect చేస్తుంది.
@with_retry() add చేస్తే ఏ function అయినా automatically retry capability వస్తుంది.
Function code change చేయక్కర్లేదు — just decorator add చేస్తే చాలు!

🇮🇳 Telugu లో Explain
Agent ki tools ఉంటాయి — search, analyze, summarize. ప్రతి tool ఒక API call. ఈ API calls ని manage చేసే system ని API Client అంటారు.

ఒక delivery company లాగా think చేయండి:
- Auth = Delivery person ID card
- Retry = Customer ఇంట్లో లేకపోతే మళ్ళీ వస్తాడు
- Parallel calls = 5 deliveries ఒకేసారి different areas కి
- Metrics = ఎన్ని deliveries success, ఎంత time పట్టింది — tracking

AgentAPIClient — Full Code

class AgentAPIClient: """ Production-grade API client for agents. Features: auth, retry, parallel, metrics. """ def __init__(self, base_url, api_key, timeout_ms=5000): self.base_url = base_url self.timeout_ms = timeout_ms self.metrics = APIMetrics() # Default headers — ప్రతి request తో వెళ్తాయి self._headers = { "Authorization": f"Bearer {api_key}", "Content-Type": "application/json", "User-Agent": "AgentClient/1.0", } @with_retry() # ← Auto-retry on failure! async def get(self, endpoint, params=None): """GET request with automatic retry""" self.metrics.total_calls += 1 response = await self._request("GET", endpoint) if response.is_success: self.metrics.successful += 1 return response async def get_many(self, endpoints): """ Multiple endpoints ని PARALLEL లో call చేయడం! asyncio.gather() use చేస్తాం. """ return await asyncio.gather( *[self.get(ep) for ep in endpoints] )
🇮🇳 Code Breakdown
__init__ — Client setup: base URL, API key, default headers set చేస్తాం
@with_retry() — ఈ decorator add చేస్తే get() method automatically retry అవుతుంది
get() — Single endpoint call + metrics tracking
get_many() — Multiple endpoints ని parallel గా call — asyncio.gather() use చేసి!

JSON Processing — OpenAI Response Parse చేయడం
🇮🇳 Telugu లో Explain
JSON = JavaScript Object Notation. ఇది internet లో data share చేసే standard format.

Python dict లాగే ఉంటుంది — {"key": "value"}
Agent ప్రతి API call నుండి JSON response receive చేస్తుంది. దాన్ని correctly parse చేయడం roz ka kaam!

Real OpenAI Response Structure

ChatGPT ని tool calling తో call చేస్తే ఈ response వస్తుంది:

# OpenAI real response (simplified) response_json = { "id": "chatcmpl-abc123", "model": "gpt-4o", "choices": [ # ← List of choices { "index": 0, "message": { # ← Nested dict! "role": "assistant", "content": None, # ← No text (tool call instead) "tool_calls": [ # ← Agent decided to use a tool! { "id": "call_xyz", "type": "function", "function": { "name": "search_web", # Tool name "arguments": '{"query": "LangGraph docs"}' # ↑ Arguments = STRING (JSON inside JSON!) } } ] }, "finish_reason": "tool_calls" } ], "usage": { "prompt_tokens": 150, "completion_tokens": 42, "total_tokens": 192 } }

Parse చేయడం — Step by Step

import json # Step 1: choices list లో first item తీయి choices = response_json["choices"] # List first_choice = choices[0] # First item # Step 2: message object తీయి message = first_choice["message"] # Step 3: tool_calls extract చేయి tool_calls = message.get("tool_calls", []) # .get() use చేయి — key లేకపోతే error రాదు! # Step 4: ప్రతి tool call process చేయి for tc in tool_calls: func = tc["function"] tool_name = func["name"] # "search_web" # IMPORTANT: arguments is a STRING, not a dict! # json.loads() తో dict లోకి convert చేయాలి arguments = json.loads(func["arguments"]) # {"query": "LangGraph docs"} print(f"Tool: {tool_name}") print(f"Args: {arguments}") # Step 5: Token usage track చేయి tokens = response_json["usage"]["total_tokens"] print(f"Tokens used: {tokens}") # 192
⚠️ Common Mistake
arguments field ఒక STRING — dict కాదు!
"arguments": '{"query": "LangGraph"}' ← ఇది string
json.loads() use చేసి dict లోకి convert చేయాలి.

ఈ mistake చాలా మంది beginners చేస్తారు — arguments["query"] direct గా access చేస్తారు → TypeError వస్తుంది!

Safe JSON Access — .get() Method

# ❌ WRONG — key లేకపోతే KeyError crash! value = data["key_that_might_not_exist"] # 💥 KeyError! # ✅ RIGHT — .get() use చేస్తే default value వస్తుంది value = data.get("key_that_might_not_exist", "default") # Key లేకపోతే "default" return అవుతుంది, crash కాదు! # ✅ Nested access — step by step choices = data.get("choices", []) if choices: message = choices[0].get("message", {}) tool_calls = message.get("tool_calls", []) else: tool_calls = [] print("No choices in response!")
💡 Pro Tip
Always .get() use చేయండి, direct [] access కాదు. API responses unpredictable — ఏ key అయినా missing ఉండవచ్చు. Production agent ఎప్పుడూ crash అవ్వకూడదు!

🚨 Critical Rule
API keys ని NEVER code లో hardcode చేయకండి!
